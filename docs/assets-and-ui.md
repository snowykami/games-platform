# 资产与 UI 指引

## 视觉资产

每个游戏应具备：

- 大厅卡片背景图：生成或提供的视觉资产，能快速表达游戏身份。
- SVG 游戏图标/favicon：16-32px 下可读、可区分。
- 可选桌面、棋盘、牌面纹理或主题 token。

没有授权资产时，可以使用生成位图作为大厅卡片背景。logo、favicon、简单符号优先使用 SVG。

不要直接使用受保护 IP 名称或美术作为主题，除非用户提供了授权资产。需要 IP 风格皮肤时，先建主题插槽，并使用原创占位主题。

## 大厅卡片

卡片需要快速传达游戏身份：

- 背景图或插画。
- 游戏标题。
- 状态 badge。
- 玩家人数和联机/本地能力。
- 使用平台标准 button variant 的主操作按钮。

不要让所有卡片退化成普通白底描边信息块。线条、颜色和按钮风格应与平台保持统一。

## 游戏 Slug 页面

进入 `/games/:slug` 后，页面视觉服从游戏本身：

- Uno 可以大胆、高对比、牌桌感强。
- 象棋可以使用木、纸、墨等语义。
- 五子棋可以突出棋盘、网格、棋子。
- 麻将可以突出牌桌、牌面、风位、牌河。
- 社交推理游戏可以突出角色、身份、讨论和投票氛围。

不要强迫 slug 页面继承大厅卡片式视觉语言。

## 布局

优先适配：

- 桌面横屏。
- 手机竖屏。

主要游戏流程应尽量避免不必要滚动。使用固定比例棋盘/牌桌区域、响应式面板和紧凑状态栏。

必须始终可见或易达：

- 当前玩家/回合高亮。
- 房间号、复制分享等在线房间动作。
- 当前阶段/状态。
- 合法动作入口。
- AI level 或 AI badge。
- 必要的错误、胜负和倒计时反馈。

移动端可以牺牲装饰、复杂动画和非关键统计，但不能丢失必要信息和主要操作方式。关键控件必须适合拇指点击，不被滚动区域或浮层遮挡。

## 控件

- 常见动作使用 lucide 图标。
- 按钮使用项目标准 variants，不临时拼接一次性颜色。
- 颜色选择使用圆形色块。
- 方向、跳过、反转等动作优先使用图标。
- 按钮、卡片、badge 内文字不能溢出。

## 动画

动画用于解释状态变化，避免用户觉得游戏“瞬间自己动了”：

- 卡牌移动、选择、出牌。
- AI 行动延迟。
- 棋子/牌/麻将牌落子。
- 胜利、将军、行动高亮。

动画应短而清晰，不拖慢游戏节奏。移动端可减少复杂动画以保证交互稳定。
